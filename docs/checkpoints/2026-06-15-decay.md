## Decisions made (canonical, verified)
- **Decay is a sibling of consolidation, not a new skill.** Folded into `cogito-consolidate`
  as "Mechanism B" so it reuses the same archive (`LESSONS-ARCHIVE.md`), probation buffer,
  load-path contract, and conservation gate. Building a second skill would have duplicated
  ~80% of that machinery; the roadmap itself groups "#4 + #5 Consolidation & decay."
- **Decay's highest-value target is project-specific lessons that leaked into the global
  brain.** The ledger loads into *every* repo, so a line about one inactive project's lint
  rule is context-rot everywhere else. That generality test is sharper than a raw importance
  floor.
- **Safety is never decayed.** `#critical` / `[I:≥9]` and destructive-command / memory-integrity
  scars are *cold by design* (rare but catastrophic) and must always load. Coldness ≠ low value
  for those.
- **Refresh-on-use = `[seen:YYYY-MM-DD]`.** Append it the instant a lesson actually fires; a
  `seen:` within the recency window makes a lesson decay-immune even if old/low-I. Honest about
  the limit: we don't timestamp every line, so position/inactivity is the fallback proxy and
  `seen:` always wins when present. (Going-forward affordance — not retrofitted with fake dates.)
- **Pass-logs/prose in the archive use `*`, never `- `.** The conservation grep (`grep -c '^- '`)
  treats every `^- ` line as a lesson, so non-lesson bullets would inflate the count. Reserve
  `- ` for real lessons only.
