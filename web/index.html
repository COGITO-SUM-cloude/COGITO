Cogito

Cogito is live — what's our mission today?

A discipline for reasoning transparently, managing finite working memory deliberately, and treating the user, the AI, and the tools as one coupled cognitive system. It doesn't make the model smarter — it makes the system smarter, by turning the right strategies from incidental into deliberate.

Explore the protocol → The ethos

Core stance

Working memory is finite. Effective capacity = buffer size × compression quality × retrieval reliability. Attack the second two; never pretend the first is infinite.

The user is the only component with guaranteed persistence across sessions. Anything that must survive a session boundary lives in a file they keep — or it's at the mercy of automatic compression.

Honesty about capability boundaries is non-negotiable. If a strategy needs the user's action, say so explicitly — never simulate success.

The session loop

01

Greet + take the mission

Open the session. Expect a vague goal — vague is the normal starting point, not a failure.

02

Interview to a spec

Keep asking until scope, success, and constraints are concrete and the assumption-check clears ~80%.

03

Red-team the mission

Critique the goal before serving it. A goal shipped un-critiqued is the most expensive error there is.

04

Build in verifiable increments

Verify each step by its real outcome — the listening port, the file in git ls-files — never a proxy.

05

Self-update on every stumble

The instant a correction or a layer-jumping bug occurs, capture the one-line lesson then and there.

06

Checkpoint + confirm

Produce a state file and ask plainly: "Is the mission accomplished?" A "no" is the most valuable signal.

07

Finish-line review

Progress → Critique → Solutions → Recommendations. Fold every fix back into durable memory.

08

The dividend

A mistake turned into a persisted lesson doesn't recur — so each loop runs leaner than the last.

Memory that compounds

Lessons are captured prospectively, one line each, and routed through the only durable write-path — a file the user owns. An empty ledger means the system is relearning its scars every session.

Ethos

This protocol is meant to be shared — anyone, human or AI, is free to learn from it and build on it. Point it at work that is honest and humane, that helps more than it harms. Prefer the efficient path to the wasteful one, the smallest sufficient compute, the change that lasts over the one rebuilt twice.

Know when to set it down. Cogito serves the work and the people, never itself. If applying it ever does more harm than good, scale it back or retire it gracefully. A good tool carries the wisdom of its own off-switch.
